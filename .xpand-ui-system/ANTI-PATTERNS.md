# Anti-Patterns

This file defines what the design system should actively avoid.
These rules are as important as the style preferences because they prevent generic, cheap, or chaotic UI outcomes.

## Global Visual Anti-Patterns

Avoid:
- boxy interfaces with weak or no rounding
- crowded layouts where everything feels stacked on top of everything else
- poor alignment
- inconsistent spacing
- no clear grid system
- weak typography hierarchy
- inconsistent component styles across the same screen
- unprofessional visual tone

Do not allow:
- random sizing where some elements feel oversized and others feel too small without reason
- overwhelming composition caused by bad proportional balance
- interfaces that feel noisy before the user even starts reading

## Dashboard Anti-Patterns

Avoid:
- overloaded dashboards
- too much information shown at once
- oversized widgets that dominate without earning attention
- inconsistent module sizing and rhythm
- weak prioritization between summary, trends, actions, and activity

Dashboard rule:
- a dashboard should help the user understand the situation quickly, not confront them with everything at once

## Form and Settings Anti-Patterns

Avoid:
- forms that are too big visually
- inconsistent field sizing
- long walls of text
- too much information presented without grouping
- sprawling settings pages that feel overwhelming

Form rule:
- forms should feel organized, controlled, and predictable

## List and Table Anti-Patterns

Avoid:
- overwhelming row design
- oversized row elements
- inconsistent spacing or action placement
- dense but unstructured tables
- layouts where scanning is difficult

List rule:
- lists should feel efficient, aligned, and easy to scan at a glance

## Tone and Branding Anti-Patterns

Avoid:
- overly playful or unprofessional color palettes
- "girly" colors when the product should feel serious and professional
- basic or generic-looking font choices
- default-looking styling that makes the product feel cheap

Tone rule:
- the system should feel premium, modern, and business-appropriate

## Summary Principle

If a screen feels:
- boxy
- crowded
- inconsistent
- oversized
- text-heavy
- poorly aligned
- weak in hierarchy

then it is moving away from the system and should be redesigned.
