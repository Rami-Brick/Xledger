# UI Review Checklist

Review every important screen against this list.

## Hierarchy
- Is the primary action obvious?
- Is the most important information visually dominant?
- Is there a clear scan path?

## Consistency
- Are spacing values repeated consistently?
- Do cards, buttons, inputs, and headings feel like one system?
- Are interaction patterns reused instead of reinvented?

## States
- Is there a defined loading state?
- Is there a useful empty state?
- Is there an error state with recovery guidance?
- Are hover, focus, active, and disabled states clear?

## Responsiveness
- Does the layout still work on mobile?
- Does the design avoid overcrowding on smaller screens?
- Do tables or dense content degrade gracefully?

## Accessibility
- Is text contrast sufficient?
- Can the interface be used by keyboard?
- Are touch targets comfortably sized?
- Is meaning conveyed by more than color alone?

## Product clarity
- Does the page explain itself quickly?
- Is the user being asked to do too many things at once?
- Is the next best action clear?

## Polish
- Are visual accents intentional rather than random?
- Are empty spaces balanced?
- Does the page feel calm and finished rather than noisy?
