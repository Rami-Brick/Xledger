# Tokens Spec

This file defines the default low-level visual tokens for the system.
These are not hard-coded brand values for every app. They are the baseline rules Claude should follow unless a project has stronger brand constraints.

## Typography

Primary direction:
- use a mix of neutral premium body typography and a slightly more expressive heading style

Reference feel:
- follow the restrained, premium tone seen in `Crextio.png`

Rules:
- body text should feel neutral, readable, and modern
- headings can carry more personality than body copy, but should remain premium and controlled
- avoid basic-looking default font combinations
- avoid overly playful or overly futuristic type choices
- title sizes should be restrained rather than oversized

Hierarchy guidance:
- page titles should feel clear and premium, not loud
- heading scale should create structure without making the interface feel heavy
- typography contrast is encouraged for lists and utility screens

## Radius

Default radius direction:
- medium to slightly large rounding

Reference feel:
- follow the rounded softness from `Crextio.png`

Rules:
- cards should use medium-to-large rounded corners
- inputs should match card rounding or be slightly rounder
- buttons should match that family and can become fully rounded when the design supports it
- avoid boxy, sharp-cornered defaults

## Spacing

Default spacing direction:
- balanced

Rules:
- dashboards should be more spacious than forms
- forms and settings can be more compact, but should not feel cramped
- use consistent spacing rhythm across sections, cards, rows, and controls
- preserve breathing room around major screen groups

## Borders and Shadows

Default direction:
- use both borders and shadows lightly

Rules:
- visible borders are allowed and encouraged
- shadows should stay soft and restrained
- cards should feel separated, but not overly detached from the page
- dark UIs should still use visible separation rather than collapsing everything into a flat surface
- separation should feel premium and controlled, not harsh

## Color Roles

General direction:
- color system depends on the app, but should remain premium and business-appropriate

Reference feel:
- for light mode, use the calm premium direction inspired by `Crextio.png`
- for dark mode, use the stronger modern direction inspired by `salesforce.webp`

Rules:
- prefer one clear accent direction per app, with supporting neutrals
- accent colors should feel intentional, not random
- light mode should feel clean, soft, and premium
- dark mode should feel modern, readable, and controlled
- avoid unprofessional or overly playful palettes

Light mode guidance:
- soft backgrounds
- premium neutral surfaces
- restrained warm or softly tinted atmosphere when appropriate

Dark mode guidance:
- deep surfaces with clear readable contrast
- controlled accent use
- visible separation through borders, tone shifts, or subtle glow

## Token Intent Summary

This system should feel:
- premium
- rounded
- restrained
- modern
- business-appropriate
- calm rather than loud

If a token decision makes the UI feel:
- boxy
- generic
- overly playful
- oversized
- harsh
- visually chaotic

then it should be reconsidered.
